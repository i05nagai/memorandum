## Service Mesh

## Terminology

* data plane
    * service instances
    * sidecar proxies
    * the interaction between them
    * the data plane handles processing too
* control plane
    * a monitoring and management layer, called the control plane.
    * creating new instances
    * terminating unhealthy or unneeded instances
